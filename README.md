# SPC-MX：三坐标测量机测量数据自动采集系统

> 自动监控三坐标测量机（CMM）输出的 Excel 测量报告，解析表头、测量明细与公差标准，并通过事务安全地写入 MySQL 数据库，为 SPC（统计过程控制）提供可靠的数据底座。

## 📖 项目背景

在机械加工质量检测场景中，三坐标测量机（CMM）会按订单/零件输出 Excel 格式的测量报告。这些报告包含尺寸特征、实测值、偏差和公差标准，是 SPC 分析的核心数据来源。

本项目通过**定时轮询目录 + 文件指纹去重 + 事务化入库**的方式，将测量报告自动、稳定地沉淀到结构化数据库中，避免人工搬运数据，保证数据可追溯、不重复、不丢失。

## ✨ 功能特性

- **目录轮询监控**：周期性扫描指定文件夹（含子目录），自动发现新增测量报告
- **多格式解析**：支持 `.xlsx` / `.xls` Excel 报告，并兼容 `.csv` 降级解析（UTF-8 / GBK 自适应）
- **文件就绪检测**：通过两次文件大小比对，确保文件已写入完成再读取，避免读到半成品
- **指纹去重**：基于文件内容 MD5 哈希判断是否已处理，已入库的文件自动跳过
- **结构化提取**：
  - 表头信息（测量计划、订单号、图号、零件号、测量时间）
  - 测量明细（特征码、实测值、偏差）
  - 公差标准（名义值、上公差、下公差）
- **事务化入库**：单文件处理全程在同一个数据库事务中完成，失败自动回滚，保证数据一致性
- **公差标准 UPSERT**：特征标准表按 `feature_code` 去重更新，保留最新版本
- **全链路日志**：记录文件处理日志（`file_process_log`）与扫描批次日志（`scan_log`），便于追踪与排查

## 🏗️ 系统架构与工作流程

```
┌─────────────┐    轮询扫描     ┌──────────────┐   就绪检测    ┌──────────────┐
│  CMM 测量机  │ ───────────▶  │  监控目录     │ ──────────▶ │  文件就绪检测  │
│  (输出报告)  │               │  E:\SPC       │              │ (大小比对)    │
└─────────────┘               └──────────────┘              └──────┬───────┘
                                                                   │
                                                                   ▼
┌──────────────┐   MD5 去重    ┌──────────────┐   解析提取    ┌──────────────┐
│  file_process │ ◀────────── │  指纹去重     │ ◀────────── │  Excel/CSV   │
│  _log (DB)   │              │  (MD5 哈希)   │              │  解析        │
└──────────────┘              └──────────────┘              └──────┬───────┘
                                                                   │
                                                                   ▼
┌──────────────┐   事务提交    ┌──────────────┐   写入明细     ┌──────────────┐
│  scan_log    │ ◀────────── │  事务入库     │ ◀────────── │  表头/明细/   │
│  (DB)        │              │  (SQLAlchemy) │              │  公差标准     │
└──────────────┘              └──────────────┘              └──────────────┘
```

**处理流程**：

1. `main.py` 启动监控器，指定监控目录（默认 `E:\SPC`）
2. `watcherdog` 循环扫描目录下所有 Excel 文件
3. 对每个文件做就绪检测（间隔 2 秒比对文件大小）
4. 计算文件 MD5，查询 `file_process_log` 判断是否已处理
5. 解析报告：提取表头、测量明细、公差标准
6. 在单个数据库事务中写入 `report_header` → `report_detail` → `feature_standard` → `file_process_log`
7. 记录本轮扫描批次日志到 `scan_log`
8. 休眠后进入下一轮扫描

## 📁 目录结构

```
spc_mx/
├── main.py                      # 程序入口，配置监控目录并启动
├── requirements.txt             # Python 依赖清单
├── app/
│   ├── watcherdog.py            # 目录监控主循环（扫描、调度、记录批次日志）
│   ├── etl/
│   │   └── cmm_parser.py        # 数据解析层（文件扫描、就绪检测、解析、提取、去重）
│   └── services/
│       └── cmm_service.py       # 业务编排层（单文件处理事务）
├── config/
│   └── db.py                    # 数据库连接配置（SQLAlchemy engine）
└── data/
    ├── TSL.csv                  # 示例数据
    └── TSL.xlsx                 # 示例报告
```

### 各模块职责

| 模块 | 职责 |
|------|------|
| `main.py` | 程序入口，指定监控目录 |
| `app/watcherdog.py` | 扫描循环、就绪检测调度、成功/跳过/失败计数、批次日志写入 |
| `app/etl/cmm_parser.py` | 文件发现、就绪检测、MD5 计算、Excel/CSV 解析、表头与明细提取、公差 UPSERT、去重查询 |
| `app/services/cmm_service.py` | 单文件处理编排，开启事务，保证多表写入的原子性 |
| `config/db.py` | SQLAlchemy engine 初始化 |

## 🗄️ 数据库设计

系统依赖 MySQL，涉及以下 5 张表：

| 表名 | 说明 |
|------|------|
| `report_header` | 测量报告表头（报告ID、测量计划、订单号、图号、零件号、测量时间） |
| `report_detail` | 测量报告明细（特征码、实测值、偏差，关联报告ID） |
| `feature_standard` | 特征公差标准（特征码、名义值、上下公差、版本号，按特征码 UPSERT） |
| `file_process_log` | 文件处理记录（文件名、路径、MD5哈希、状态），作为去重依据 |
| `scan_log` | 扫描批次日志（扫描时间、目录、总数/新增/跳过/失败数） |

## 🚀 安装与配置

### 1. 环境准备

```bash
# 克隆仓库
git clone https://github.com/Jeffery-source/spc_mx.git
cd spc_mx

# 安装依赖
pip install -r requirements.txt
```

### 2. 数据库配置

修改 `config/db.py` 中的数据库连接串，指向你自己的 MySQL 实例：

```python
from sqlalchemy import create_engine

engine = create_engine(
    "mysql+pymysql://<用户名>:<密码>@<主机>:<端口>/spc_mx"
)
```

> ⚠️ **安全提醒**：当前仓库的 `config/db.py` 中硬编码了数据库账号密码，存在凭据泄露风险。建议：
> - 使用环境变量管理敏感信息，避免提交到代码仓库
> - 若凭据已泄露，请及时修改数据库密码
> - 将 `config/db.py` 加入 `.gitignore`，或改为读取 `.env` 配置

### 3. 创建数据库表

根据上文「数据库设计」在目标库中建好 `report_header`、`report_detail`、`feature_standard`、`file_process_log`、`scan_log` 五张表。

### 4. 配置监控目录

修改 `main.py` 中的 `folder` 变量，指向 CMM 测量机输出报告的目录：

```python
folder = r"E:\\SPC"  # 改为你的实际目录
```

## 📖 使用方法

```bash
python main.py
```

程序启动后会持续运行，每轮扫描间隔约 10000 秒（可在 `app/watcherdog.py` 中调整 `time.sleep()` 的值）。

控制台会输出扫描进度与处理结果：

```
扫描目录: E:\SPC
文件已处理，跳过: E:\SPC\TSL.xlsx
入库成功
```

## ⚠️ 注意事项与改进建议

1. **数据库凭据硬编码**：`config/db.py` 明文包含账号密码，建议改用环境变量或配置文件（详见上文「数据库配置」）
2. **临时文件过滤**：Excel 打开时产生的 `~$` 临时文件，当前在 `process_file` 之后才判断跳过，建议在 `scan_excel_files` 阶段即过滤，避免无谓的 MD5 计算与解析
3. **异常处理**：`cmm_service.py` 中事务失败会 `raise`，而 `watcherdog.py` 未捕获该异常，可能导致整个程序中断。建议在 watcher 循环内 try/except 包裹单文件处理，保证单文件失败不影响整体运行
4. **扫描间隔**：`time.sleep(10000)` 约 2.7 分钟，可视业务实时性需求调整
5. **文件路径分隔符**：代码中 `file_path.split("/")[-1]` 提取文件名，在 Windows 下路径分隔符为 `\`，可能导致文件名截取异常，建议改用 `os.path.basename()`
